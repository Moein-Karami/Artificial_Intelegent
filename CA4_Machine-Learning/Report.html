Normalize: این متد متغیر های عددی را به یک عدد بین ۰ و ۱ تبدیل می‌کند که معادله این تبدیل برابر است با: $$ x_{new} = \frac{x_{old} - x_{min}}{x_{max} - x_{min}}$$
Standardization: در این متد متغیر ها به شکلی تغییر می‌کنند که میانگین آن ها برار ۰ و انحراف معیارشان برابر ۱ باشد. $$x_{new} = \frac{x_{old} - \mu}{\sigma}$$

5)

۱. Label Encoding or Ordinal Encoding: در این روش به هر دسته یک عدد نسبت داده می‌شود، این متد بیشتر در داده هایی استفاده میشود که دسته ها در آن ترتیب دارند.
۲. One Hot Encoding: فرض کنید یک ستون شامل داده دسته ای است که می‌تواند شامل x مقدار مختلف باشد.
در این روش این ستون داده را حذف کرده و x ستون اضافه می‌کنیم و اگر در ستون اصلی این متغیر شامل دسته ی iام بود در خانه ی مربوط به این داده و ستون iام که جدید اضافه شده، ۱ می‌گذاریم و باقی خانه های مربوط به این دیتا در ستون های اضافه شده ۰ قرار می‌دهیم.

من از متد اول استفاده می‌کنم چون متد دوم در داده های دسته ای که می‌توانند مقادیر مختلف زیادی داشته باشند(مانند ستون اول و دوم دیتای ما) بسیار بد عمل می‌کند و همچنین متد اول هم سربع تر است و هم فضای کمتری اشغال می‌کند

1)

مقدار p را برابر 10 در نظر می‌گیریم.

بلی داده ها باید به صورت اتفاقی تقسیم شوند، چون در غیر این صورت مثلا اگر ۱۰ درصد داده اول را برای داده تست انتخاب کنیم، در این صورت تمام تست دیتای ما موسیقی جاز هستند و ما نمیتوانیم به درستی مدل خود را ارزیابی کنیم زیرا هیچ داده ی دیگری از ژانر های دیگر برای تست کردن نداریم.

بهتر است که نسبت ژانر ها به کل در هر دو داده ی تست و هم ترین برابر باشد.
برابر بودن این نسبت در داده ترین باعث می‌شود که مدل آموزش دیده ی ما بایاس نداشته باشد، به عبارت دیگر اگر از یک ژانر تعداد زیادی داده در ترین دیتا داشته باشیم و از ژانر دیگه ای مقدار کمی دیتا داشته باشیم، پیشبینی های ما بایاس دار می‌شوند و بیشتر به سمت ژانری می‌روند که سمپل های زیادی از آن دیده ایم و کمتر ژانری را پیش‌بینی می‌کند که دیتای کمی از آن دیده.
برابر بودن این نسبت در تست دیتا باعث می‌شود که ما به درستی بتوانیم مدل خود را ارزیابی کنیم، چون کم بودن دیتا از یک ژانر خاص باعث می‌شود ما نتوانیم به درستی قدرت مدل خود را در پیشبینی این ژانر را ارزیابی کنیم.

متغیر stratify در تابع train_test_split یک آرایه است که به صورت پیشفرض None است و اگر None نباشد، دیتا بر اساس این آرایه تقسیم می‌شود.

best_i = 1
for i in range(1, 50):
    knn = KNeighborsClassifier(n_neighbors = i)

        best_i, best = i, tmp

6)
Recall : $$ Recall = \frac{Correct\ Detected\ Genre}{Total\ Genre}$$ این معیار محاسبه می‌کند که چند درصد از داده هایی که یک ژانر خاص را داشته اند، به درستی این ژانر را برای آن دیتا پیش‌بینی کرده ایم، ولی اهمیتی نمی‌دهد که چند دیتای اشتباه دیگر را در این ژانر طبقه بندی کرده ایم.
به بیان دیگر False Positive در این معیار تاثیری ندارد.

Precision: $$Preciion = \frac{Correct\ Detected\ Genre}{All Detected Genre}$$ این معیار محاسبه می‌کند چند درصد از داده هایی که برای آن ها یک ژانر خاص پیش‌بینی شده است، به درستی پیش بینی شده اند. ولی اهمیتی نمی‌دهد چند دیتا از این ژانر بوده‌اند ولی ژانر آن ها چیز دیگری تشخیص داده شده است.
به عبارت دیگر False Negative در این معیار تاثیری ندارد.

Accuracy: $$Accuracy = \frac{Correct\ Detected}{Total}$$ در این معیار به درصد تشخیص های درست اهمیت داده میشود و به طور شهودی برابر با میانگین دو معیار بالاست.

F1 Score: $$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$ این معیار میانگین هارمونیک دو معیار اولیه است در نتیجه اگر یکی از این معیار ها خیلی پایین باشد، این معیار نیز به آن متمایل می‌شود و امتیاز کمی می‌گیریم.

def calculate_accuracy(true_values, predicted_values):
    true_gues= 0;
    for i, j in zip(true_values, predicted_values):
        true_gues += int(i == j)
    return true_gues / len(true_values)

def calculate_precision(true_values, predicted_values, label):
    correct_gues = 0
    all_gues = 0
    for i, j in zip(true_values, predicted_values):
        if j == label:
            all_gues += 1
            if i == j:
                correct_gues += 1
    return correct_gues / all_gues

def calculate_recall(true_values, predicted_values, label):
    correct_gues = 0
    in_label = 0
    for i, j in zip(true_values, predicted_values):
        if i == label:
            in_label += 1
            if i == j:
                correct_gues += 1
    return correct_gues / in_label

def calculate_f1(true_values, predicted_values, label):
    recall = self.calculate_recall(true_values, predicted_values, label)
    precision = self.calculate_precision(true_values, predicted_values, label)
    if (recall + precision == 0):
        return 0
    return 2 * (recall * precision) / (recall + precision)
